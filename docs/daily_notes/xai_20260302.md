## Commit `f68fbad` — search: city-pill contrast fix, maximizable panels, filter toolbar

### 1. City-pill dark mode contrast fix

**Problem:** "Hessen › Frankfurt am Main" pill nearly invisible — bright font on
bright background in dark mode.

**Root cause:** A pre-existing `.city-pill` rule at line ~5499 used `!important`
on `background: #e0f2fe` with no dark-mode override. This overrode our new
`[data-theme="dark"] .filter-pill.city-pill` rules.

**Fix:**
- Removed the old `.city-pill { background: #e0f2fe !important }` rule
- Added explicit `.filter-pill.city-pill` colors for both themes:
  - Light: `#e8f0fe` bg, `#1a56db` text
  - Dark: `#1a2744` bg, `#93bbfc` text (~7:1 contrast ratio)

### 2. Maximizable panels

Added ⛶ maximize button to all 3 search panels (left, map, right) + results
grid below. Click toggles full-width view; click ⊗ to restore.

**Implementation:**
- `.panel-maximize-btn` — absolute positioned, opacity 0→1 on panel hover
- `.search-panels.has-maximized` — switches grid to `1fr` single column
- Non-maximized siblings get `display: none !important` (needed because
  `.panel-map { display: flex }` at line 5505 wins by cascade otherwise)
- Results grid maximizes as fixed overlay (`z-index: 900`)
- Calls `map.invalidateSize()` after toggle for proper Leaflet rendering

### 3. Filter management toolbar

Replaced single "Save search" button with 4-button toolbar:

| Button | Action |
|--------|--------|
| 👤 From profile | Clears filters, re-infers from profile scope |
| 🗑 Clear all | Empties all state arrays, resets map to Germany center |
| 📂 Load saved | Loads previously saved search from server |
| 💾 Save search | Persists current filters to server (unchanged logic) |

Added i18n keys for all new buttons in both `en.json` and `de.json`.
