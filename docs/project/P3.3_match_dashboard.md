## Components

### Match Card

- Recommendation badge (APPLY/SKIP)
- Match score (percentage)
- Job title + company
- Location
- Top matching skills OR top concern
- Action buttons

### Filters

- All / Recommended / Skipped
- Score range (slider)
- Location (dropdown)
- Date (last 7 days, 30 days, all)

### Actions

| Action | Effect |
|--------|--------|
| View Report | Opens full transparency report |
| View Job | Links to original posting |
| Mark Applied | Updates `user_applied=true` |
| Dismiss | Hides from list |
| Rate (1-5) | Updates `user_rating` |

## Acceptance Criteria

- [x] Dashboard loads with user's matches
- [x] Cards show score, title, company, location
- [x] APPLY vs SKIP clearly distinguished
- [x] Filters work (recommendation, score, date)
- [x] Actions update database
- [x] Mobile responsive

## Sandy's Notes

The dashboard IS the product. Spend time here.

Sort by recommendation first (APPLY at top), then by score. Users want action items, not a score leaderboard.

The "high score but SKIP" cases (Finance Advisor at 89%) are the interesting ones. Surface the concern prominently.
